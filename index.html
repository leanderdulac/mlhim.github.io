Multi-Level Healthcare Information Modelling (MLHIM) is a set of specifications that combine XML Schema, RDF and OWL in an innovative way, to achieve semantic interoperability in Healthcare IT.

Multi-Level Information Modelling (MLHIM) is an approach to programming computer systems that need to transmit data with unambiguous, shared meaning between different implementations. We advocate the use of an abstract framework embodying all the standards and specifications of the data to be transmitted. Although this framework will often be human-readable, user-friendly interfaces that represent the data accurately, such as for a GUI, should also be defined.

Do you know why interoperability is hard?

With traditional software design approaches, the semantics are locked up in the application code and database schema. Today, healthcare Big Data is scattered around a constellation of data silos. Having access to the semantics of the data is essential for clinical and epidemiological decision support. This is the challenge that we address with MLHIM.

MLHIM is first a concept. Based on this concept, there are frameworks, applications and tools used to develop health care applications and the governance of knowledge artifacts using a multi-level information model. MLHIM is an implementation independent model. The reference implementation is expressed in XML Schema Language 1.1

MLHIM is multi-level in regards to a Reference Model that is developed in software with the conceptual data models implemented using XML Schemas that express constraints on the Reference Model.

The MLHIM XML Schemas can then be easily shared between systems so that the semantic context is made available to the data in any conforming application. The XML Schema data models can be aggregated in order to form queries, messages, screen forms, reports, i.e., any type of healthcare application.

It is important, in the context of the current global world of healthcare IT, to describe what MLHIM *IS NOT*.

MLHIM is not an EMR/EHR specification. MLHIM is not a messaging specification.

MLHIM is the foundation of these and other healthcare IT applications. It is the foundation that provides the capability for full semantic interoperability via open standards and open specifications.
